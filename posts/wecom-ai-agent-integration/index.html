📱 企业微信新版本 AI 能力概览

核心 AI 功能

企业微信新版本的 AI 能力主要围绕以下几个方面：

功能模块能力描述应用场景
智能文档文档内容理解、自动生成、智能编辑会议纪要、项目文档、周报
智能表格数据结构化管理、自动填充、公式推荐任务跟踪、数据报表、项目管理
智能会议会议预约、成员管理、会议记录团队例会、项目评审
智能日程日程安排、闲忙分析、冲突检测会议安排、时间管理
消息智能消息分类、自动回复、内容提取客户沟通、内部协作

AI 能力的底层支撑

这些 AI 功能的背后，是企业微信开放的 MCP (Model Context Protocol) 接口体系。通过标准化的 API，外部智能体可以：

  • 读取和编辑文档内容
  • 管理表格的结构和数据
  • 查询和操作用户通讯录
  • 发送消息和多媒体内容

🔌 OpenClaw 与企业微信的对接架构

什么是 OpenClaw？

OpenClaw 是一个 AI 智能体运行框架，它提供了：

  • 技能系统：模块化的能力封装
  • 记忆系统：长期记忆和上下文管理
  • 通道集成：对接企业微信、Telegram 等消息平台
  • 任务调度：定时任务和自动化工作流

对接架构

┌─────────────────┐         ┌─────────────────┐
│   企业微信 API   │ ←────→  │  OpenClaw       │
│   (MCP Server)  │  HTTP   │  WeCom Agent    │
└─────────────────┘         └─────────────────┘
         ↑                           ↑
         │                           │
    云文档/表格                   技能系统
    通讯录/会议                   记忆系统
    日程/消息                     任务调度

核心组件

  1. wecom_mcp：企业微信 MCP 工具，所有 API 调用的统一入口
  2. 技能系统：将不同功能封装为独立技能（Skill）
  3. 前置检查：确保工具权限和白名单配置正确

前置条件检查

在首次调用任何企业微信 API 前，需要执行前置检查：

# 检查工具权限配置
wecom-preflight

这个检查会验证：

  • MCP 工具是否在白名单中
  • Profile 配置是否正确
  • 是否需要修复配置并重启 Gateway

📄 智能体操作云文档的完整指南

文档类型识别

企业微信有三种文档类型，URL 格式不同，读取接口也不同：

URL 模式文档类型读取接口
https://doc.weixin.qq.com/doc/*普通文档get_doc_content
https://doc.weixin.qq.com/smartsheet/*智能表格get_doc_content
https://doc.weixin.qq.com/smartpage/*智能主页smartpage_export_task

关键规则：先看 URL 路径，再选择正确的接口！

1. 读取文档内容

读取文档采用异步轮询机制：

# 第一步：发起导出任务
wecom_mcp call doc get_doc_content '{"docid": "DOCID", "type": 2}'

# 返回示例
{
  "errcode": 0,
  "task_id": "task_xxx",
  "task_done": false
}

# 第二步：轮询直到完成
wecom_mcp call doc get_doc_content '{"docid": "DOCID", "type": 2, "task_id": "task_xxx"}'

# 最终返回
{
  "errcode": 0,
  "task_done": true,
  "content": "# 文档标题\n\n文档内容..."
}

2. 创建新文档

# 创建普通文档
wecom_mcp call doc create_doc '{"doc_type": 3, "doc_name": "项目周报"}'

# 创建智能表格
wecom_mcp call doc create_doc '{"doc_type": 10, "doc_name": "任务跟踪表"}'

# 返回示例
{
  "errcode": 0,
  "docid": "doc_xxx",
  "url": "https://doc.weixin.qq.com/doc/xxx"
}

注意：docid 仅在创建时返回，务必妥善保存！

3. 编辑文档内容

用 Markdown 格式覆写文档正文：

wecom_mcp call doc edit_doc_content '{
  "docid": "DOCID",
  "content": "# 项目周报\n\n## 本周进展\n\n- 完成需求评审\n- 开始开发...",
  "content_type": 1
}'

4. 创建智能主页

智能主页支持多个子页面：

wecom_mcp call doc smartpage_create '{
  "title": "项目概览",
  "pages": [
    {
      "page_title": "需求文档",
      "content_type": 1,
      "page_filepath": "/path/to/requirements.md"
    },
    {
      "page_title": "技术方案",
      "content_type": 1,
      "page_filepath": "/path/to/design.md"
    }
  ]
}'

注意事项：

  • content_type 必须与文件实际内容匹配（.md 文件传 1）
  • 每个子页面不超过 10MB

5. 导出智能主页内容

智能主页导出是异步的两步过程：

# 第一步：发起导出任务
wecom_mcp call doc smartpage_export_task '{"docid": "DOCID", "content_type": 1}'

# 第二步：轮询获取结果
wecom_mcp call doc smartpage_get_export_result '{"task_id": "TASK_ID"}'

# 当 task_done 为 true 时，读取 content_filepath 指向的文件

📊 智能体操作智能表格的完整指南

智能表格是企业微信的杀手级功能，智能体可以：

  • 查询和修改表结构（子表、字段）
  • 增删改查表格数据（记录）

表格结构管理

1. 查询子表列表

wecom_mcp call doc smartsheet_get_sheet '{"docid": "DOCID"}'

# 返回示例
{
  "errcode": 0,
  "sheets": [
    {
      "sheet_id": "sheet_xxx",
      "title": "任务列表",
      "type": "SHEET_TYPE_NORMAL"
    }
  ]
}

2. 查询字段信息

wecom_mcp call doc smartsheet_get_fields '{"docid": "DOCID", "sheet_id": "SHEETID"}'

# 返回示例
{
  "errcode": 0,
  "fields": [
    {
      "field_id": "field_xxx",
      "field_title": "任务名称",
      "field_type": "FIELD_TYPE_TEXT"
    },
    {
      "field_id": "field_yyy",
      "field_title": "负责人",
      "field_type": "FIELD_TYPE_USER"
    }
  ]
}

3. 添加新字段

wecom_mcp call doc smartsheet_add_fields '{
  "docid": "DOCID",
  "sheet_id": "SHEETID",
  "fields": [
    {"field_title": "优先级", "field_type": "FIELD_TYPE_SINGLE_SELECT"},
    {"field_title": "截止日期", "field_type": "FIELD_TYPE_DATE"}
  ]
}'

常用字段类型：

  • FIELD_TYPE_TEXT：文本
  • FIELD_TYPE_NUMBER：数字
  • FIELD_TYPE_DATE：日期
  • FIELD_TYPE_USER：成员
  • FIELD_TYPE_SINGLE_SELECT：单选
  • FIELD_TYPE_CHECKBOX：复选框

表格数据管理

1. 查询记录

wecom_mcp call doc smartsheet_get_records '{"docid": "DOCID", "sheet_id": "SHEETID"}'

# 返回示例
{
  "errcode": 0,
  "records": [
    {
      "record_id": "rec_xxx",
      "values": {
        "任务名称": [{"type": "text", "text": "完成需求文档"}],
        "优先级": [{"text": "高"}],
        "负责人": [{"user_id": "ZC3609", "name": "黄永晔"}]
      }
    }
  ]
}

2. 添加记录

关键注意：成员（USER）类型字段需要 user_id，不能直接用姓名！

# 先查询用户 userid
wecom_mcp call contact get_userlist '{}'

# 然后添加记录
wecom_mcp call doc smartsheet_add_records '{
  "docid": "DOCID",
  "sheet_id": "SHEETID",
  "records": [
    {
      "values": {
        "任务名称": [{"type": "text", "text": "完成需求文档"}],
        "优先级": [{"text": "高"}],
        "负责人": [{"user_id": "ZC3609"}]
      }
    }
  ]
}'

3. 更新记录

wecom_mcp call doc smartsheet_update_records '{
  "docid": "DOCID",
  "sheet_id": "SHEETID",
  "key_type": "CELL_VALUE_KEY_TYPE_FIELD_TITLE",
  "records": [
    {
      "record_id": "rec_xxx",
      "values": {
        "优先级": [{"text": "紧急"}]
      }
    }
  ]
}'

4. 删除记录

wecom_mcp call doc smartsheet_delete_records '{
  "docid": "DOCID",
  "sheet_id": "SHEETID",
  "record_ids": ["rec_xxx", "rec_yyy"]
}'

警告：删除操作不可逆，请谨慎使用！

💡 实战案例：自动化任务管理系统

场景描述

团队需要一个自动化的任务管理系统：

  1. 每天自动创建任务跟踪表格
  2. 从会议纪要中提取任务项
  3. 自动分配给对应负责人
  4. 定期更新任务状态

实现步骤

步骤 1：创建任务跟踪表

# 创建智能表格
wecom_mcp call doc create_doc '{"doc_type": 10, "doc_name": "任务跟踪表"}'

# 获取返回的 docid，保存备用

步骤 2：定义表格结构

# 查询默认子表
wecom_mcp call doc smartsheet_get_sheet '{"docid": "DOCID"}'

# 添加字段
wecom_mcp call doc smartsheet_add_fields '{
  "docid": "DOCID",
  "sheet_id": "SHEETID",
  "fields": [
    {"field_title": "任务名称", "field_type": "FIELD_TYPE_TEXT"},
    {"field_title": "描述", "field_type": "FIELD_TYPE_TEXT"},
    {"field_title": "负责人", "field_type": "FIELD_TYPE_USER"},
    {"field_title": "优先级", "field_type": "FIELD_TYPE_SINGLE_SELECT"},
    {"field_title": "截止日期", "field_type": "FIELD_TYPE_DATE"},
    {"field_title": "状态", "field_type": "FIELD_TYPE_SINGLE_SELECT"},
    {"field_title": "创建时间", "field_type": "FIELD_TYPE_CREATE_TIME"},
    {"field_title": "最后更新", "field_type": "FIELD_TYPE_MODIFY_TIME"}
  ]
}'

步骤 3：自动添加任务

# 从会议纪要提取任务后，批量添加
wecom_mcp call doc smartsheet_add_records '{
  "docid": "DOCID",
  "sheet_id": "SHEETID",
  "records": [
    {
      "values": {
        "任务名称": [{"type": "text", "text": "完成 API 设计"}],
        "描述": [{"type": "text", "text": "设计 RESTful API 接口规范"}],
        "负责人": [{"user_id": "ZC3609"}],
        "优先级": [{"text": "高"}],
        "截止日期": [{"date": "2026-04-30"}],
        "状态": [{"text": "进行中"}]
      }
    }
  ]
}'

步骤 4：定时更新状态

使用 OpenClaw 的定时任务功能，每天检查并更新任务状态：

# HEARTBEAT.md 配置
### 任务状态每日更新
- 时间：每天 9:00
- 操作：
  1. 查询所有"进行中"的任务
  2. 检查是否超过截止日期
  3. 超期任务标记为"延期"
  4. 发送企业微信通知给负责人

🚀 最佳实践与注意事项

1. 权限管理

  • 首次使用前执行 wecom-preflight 检查
  • 确保工具已加入白名单
  • 敏感操作（删除、批量更新）需要二次确认

2. 错误处理

# 通用错误处理逻辑
if errcode != 0:
    # 可重试的错误，重试 1 次
    if errcode in [500, 502, 503]:
        retry()
    # 文档类型不匹配，切换接口重试
    elif errcode == 851002:
        switch_interface_and_retry()
    # 其他错误，展示给用户
    else:
        show_error_to_user(errcode, errmsg)

3. 性能优化

  • 批量操作控制在 500 条记录以内
  • 异步接口采用轮询机制，避免频繁请求
  • 缓存 docid、sheet_id、field_id 等标识符

4. 数据安全

  • 删除操作不可逆，操作前备份
  • 成员字段使用 userid，不直接存储姓名
  • 敏感信息不记录到日志

5. 用户体验

  • 异步操作显示进度提示
  • 错误信息翻译成用户友好的语言
  • 复杂操作提供确认步骤
