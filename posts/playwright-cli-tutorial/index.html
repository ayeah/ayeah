Playwright CLI 完全指南：从入门到百度搜索实战

全面介绍 Playwright CLI 的使用方法、高级技巧，并通过百度搜索实战演练，帮助你快速掌握浏览器自动化测试

前言

Playwright CLI 是 Microsoft 开发的浏览器自动化工具的命令行接口，专为测试、脚本编写和 AI 代理设计。它支持 Chromium、Firefox 和 WebKit 三大主流浏览器引擎，可在 Linux、macOS 和 Windows 上运行。

本文将带你从安装配置开始，逐步掌握 Playwright CLI 的核心功能，并通过百度搜索自动化实战，让你快速上手。

一、安装与配置

1.1 全局安装（推荐）

npm install -g @playwright/cli@latest

1.2 项目本地安装

# 初始化 Playwright 项目
npm init playwright@latest

# 或手动安装
npm install @playwright/test
npx playwright install

1.3 安装浏览器

# 安装所有支持的浏览器
playwright install

# 安装特定浏览器
playwright install chromium
playwright install firefox
playwright install webkit

# Linux 系统安装带依赖的浏览器
playwright install --with-deps chromium

1.4 验证安装

playwright --version

二、核心使用方法

2.1 代码生成器（Codegen）

Codegen 是 Playwright 最强大的功能之一，可以记录你的浏览器操作并自动生成测试代码。

# 打开代码生成器
npx playwright codegen

# 指定目标网站
npx playwright codegen https://www.baidu.com

# 指定浏览器
npx playwright codegen --browser chromium https://example.com

使用技巧：

  • 在浏览器中的所有点击、输入操作都会被记录
  • 可实时查看生成的代码
  • 支持 Python、JavaScript、TypeScript、Java、C# 多种语言

2.2 运行测试

# 运行所有测试
npx playwright test

# 运行特定测试文件
npx playwright test tests/example.spec.ts

# 运行特定测试（通过名称匹配）
npx playwright test -g "登录"

# 有头模式（显示浏览器界面）
npx playwright test --headed

# 指定浏览器
npx playwright test --project=chromium

# 调试模式
npx playwright test --debug

2.3 查看测试报告

# 生成 HTML 报告
npx playwright test --reporter=html

# 打开报告
npx playwright show-report

2.4 追踪查看器（Trace Viewer）

Trace Viewer 可以查看测试执行的完整时间线，包括 DOM 快照、网络请求、控制台日志等。

# 运行测试并生成追踪
npx playwright test --trace on

# 查看追踪文件
npx playwright show-trace trace.zip

三、高级技巧

3.1 智能定位器策略

// ✅ 推荐：使用用户视角的定位器
page.getByRole('button', { name: '提交' })
page.getByLabel('用户名')
page.getByPlaceholder('请输入邮箱')
page.getByTestId('submit-btn')
page.getByText('欢迎回来')

// ❌ 避免：脆弱的 CSS/XPath 选择器
page.locator('#submit-btn')  // 不推荐
page.locator('div > span:nth-child(3)')  // 避免

3.2 自动等待与断言

// Playwright 自动等待元素可操作
await page.getByRole('button').click()

// 自动重试的断言
await expect(page.locator('.status')).toHaveText('成功')

// 显式等待
await page.waitForLoadState('networkidle')
await page.waitForTimeout(1000)  // 尽量避免使用

3.3 处理弹窗和对话框

// 处理 alert/confirm
page.on('dialog', async dialog => {
  console.log(dialog.message())
  await dialog.accept()
})

// 处理新标签页
const [newPage] = await Promise.all([
  context.waitForEvent('page'),
  page.getByText('新窗口').click()
])

3.4 文件上传下载

// 文件上传
await page.locator('input[type="file"]').setInputFiles('path/to/file.pdf')

// 文件下载
const [download] = await Promise.all([
  page.waitForEvent('download'),
  page.getByText('下载文件').click()
])
await download.saveAs('downloads/my-file.pdf')

3.5 认证状态复用

// playwright.config.js
module.exports = {
  use: {
    storageState: 'auth.json'  // 保存认证状态
  }
}

// 登录一次后保存状态，后续测试直接复用

四、实战演练：百度搜索自动化

4.1 场景描述

模拟用户在百度搜索"Playwright 教程"，验证搜索结果页面正常加载。

4.2 完整代码示例

// tests/baidu-search.spec.js
const { test, expect } = require('@playwright/test');

test.describe('百度搜索测试', () => {
  
  test.beforeEach(async ({ page }) => {
    // 每个测试前访问百度首页
    await page.goto('https://www.baidu.com');
  });

  test('搜索 Playwright 教程', async ({ page }) => {
    // 1. 定位搜索框并输入关键词
    const searchInput = page.locator('#kw');
    await searchInput.fill('Playwright 教程');
    
    // 2. 点击搜索按钮
    const searchButton = page.locator('#su');
    await searchButton.click();
    
    // 3. 等待搜索结果加载
    await page.waitForLoadState('networkidle');
    
    // 4. 验证搜索结果页面
    await expect(page).toHaveURL(/wd=Playwright/);
    
    // 检查搜索结果容器存在
    const results = page.locator('#content_left');
    await expect(results).toBeVisible();
    
    // 验证至少有一个搜索结果
    const resultItems = page.locator('.result.c-container');
    await expect(resultItems.first()).toBeVisible();
    
    // 截图保存
    await page.screenshot({ path: 'screenshots/baidu-search-result.png' });
    
    console.log('✓ 百度搜索测试通过');
  });

  test('搜索框快捷键测试', async ({ page }) => {
    const searchInput = page.locator('#kw');
    
    // 输入关键词后按回车
    await searchInput.fill('自动化测试');
    await searchInput.press('Enter');
    
    // 等待页面加载
    await page.waitForLoadState('domcontentloaded');
    
    // 验证搜索结果
    await expect(page.locator('#content_left')).toBeVisible();
    
    console.log('✓ 快捷键搜索测试通过');
  });

  test('搜索建议功能测试', async ({ page }) => {
    const searchInput = page.locator('#kw');
    
    // 输入部分关键词
    await searchInput.fill('Playw');
    
    // 等待搜索建议出现
    const suggestions = page.locator('.bdsug-li');
    await expect(suggestions.first()).toBeVisible({ timeout: 5000 });
    
    // 获取建议数量
    const count = await suggestions.count();
    console.log(`搜索建议数量：${count}`);
    
    // 点击第一个建议
    if (count > 0) {
      await suggestions.first().click();
      await page.waitForLoadState('networkidle');
      await expect(page.locator('#content_left')).toBeVisible();
    }
    
    console.log('✓ 搜索建议测试通过');
  });
});

4.3 配置文件

// playwright.config.js
module.exports = {
  testDir: './tests',
  timeout: 30000,
  expect: {
    timeout: 5000
  },
  retries: 1,
  workers: 2,
  reporter: [
    ['html', { outputFolder: 'playwright-report' }],
    ['list']
  ],
  use: {
    viewport: { width: 1920, height: 1080 },
    actionTimeout: 10000,
    screenshot: 'only-on-failure',
    trace: 'retain-on-failure',
    launchOptions: {
      slowMo: 500,  // 慢动作模式，便于观察
    }
  },
  projects: [
    {
      name: 'chromium',
      use: { browserName: 'chromium' }
    },
    {
      name: 'firefox',
      use: { browserName: 'firefox' }
    },
    {
      name: 'webkit',
      use: { browserName: 'webkit' }
    }
  ]
};

五、总结与最佳实践

5.1 核心优势

特性说明
跨浏览器一套代码支持 Chromium、Firefox、WebKit
自动等待无需手动设置 sleep，减少不稳定测试
弹性定位使用语义化选择器，降低维护成本
调试友好Trace Viewer、Codegen、VS Code 扩展
AI 集成MCP 服务器支持，适合 AI 代理

5.2 最佳实践清单

  1. 优先使用语义化定位器

    • getByRole() > getByText() > CSS 选择器

  2. 避免硬编码等待

    • 使用 waitForLoadState() 代替 waitForTimeout()

  3. 测试隔离

    • 每个测试独立，不依赖其他测试状态

  4. 合理使用截图和追踪

    • 失败时自动截图，便于问题排查

  5. 并行执行优化

    • CI 环境使用分片，加快执行速度

  6. 认证状态复用

    • 登录状态保存，避免重复登录

5.3 常见问题解决

问题解决方案
元素找不到检查定位器，使用 Codegen 重新生成
测试不稳定增加自动等待，避免硬编码延迟
执行太慢启用并行，减少不必要的等待
内存占用高限制 workers 数量，及时清理上下文
